## 场景入场语法糖提案（第一版）

这份提案用于解决一个高频写法痛点：

当元素在前一个场景不存在、在当前场景首次出现时，开发者不想为了过渡效果手动回到前一个场景补一份"画外状态"。

目标是提供一个轻量语法糖，让 Scene 继续只描述状态，但在运行时可以自动补出过渡起点。

---

### 一、设计原则

1. 语法糖入口放在 Scene API 层（setState 的 options 参数）
2. Scene 仍然只存状态定义，不拥有元素实例
3. 运行时补全前态，不回写上一场景状态表
4. 显式配置优先，默认值只在完全缺省时生效
5. 正放与倒放使用同一套规则，保证可逆

---

### 二、API 形态

```js
scene.setState(element, state, {
  entrance: true
})
```

也支持显式前态：

```js
scene.setState(element, state, {
  entrance: {
    from: {
      opacity: 0,
      transform: { y: 40 }
    }
  }
})
```

---

### 三、语义规则

#### 1) 正放（fromScene -> toScene）

当元素在 toScene 有状态、fromScene 没状态，且 toScene 为该元素启用 entrance：

- 若 entrance.from 存在：以 entrance.from 作为前态
- 若 entrance.from 不存在：前态默认补为 `{ opacity: 0 }`

#### 2) 倒放（toScene -> fromScene）

当从后往前播放，若目标场景缺失状态而当前场景启用了 entrance：

- 使用与正放相同的 entrance.from/默认规则构造回退目标态
- 保证体验上与正放逻辑对称

#### 3) 显式优先

如果 entrance.from 显式写了某字段（如 opacity、transform），该字段不使用默认值覆盖。

---

### 四、默认值约定（第一版）

第一版只提供一个可见性兜底：

- 当 entrance.from 完全未定义 opacity 时，默认 `opacity: 0`

说明：

- 这符合当前对"入场"的直觉定义（先不可见再出现）
- 不隐式改动 layout、transform，避免超出预期

---

### 五、推荐写法

```js
const scene1 = new Scene("scene1")
const scene2 = new Scene("scene2")

scene2.setState(
  title,
  {
    layout: {
      mode: "absolute",
      left: "50%",
      top: "42%",
      width: "auto",
      height: "auto",
      anchorX: 0.5,
      anchorY: 0.5,
    },
    opacity: 1,
  },
  {
    entrance: true,
  }
)
```

---

### 六、运行时职责边界

- Scene：存储 state + entrance 元数据
- Presentation：根据 fromScene/toScene 解析有效的过渡 fromState/toState
- Renderer：消费解析结果并执行插值

这与当前架构保持一致：

- 布局计算与渲染依旧在 Renderer/Layout 层
- Scene 只负责状态声明，不引入 DOM 逻辑

---

### 七、非目标范围（第一版）

1. 不做 timeline keyframe 系统
2. 不做复杂缓动曲线 DSL
3. 不做编译期状态展开
4. 不自动改写历史场景数据

---

### 八、后续可扩展方向

1. entrance.from 支持按字段策略合并
2. 支持指定继承来源（上一场景 / 指定场景）
3. 提供开发态调试输出（展示推导后的 fromState/toState）
4. 在场景规模较大时结合分片加载机制
