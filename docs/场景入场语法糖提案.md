## 场景入场与出场语法糖提案（第一版）

本提案统一定义 `entrance` 与 `exit` 的使用方式，并说明正放/倒放时的解析规则。

---

### 一、设计原则

1. 语法糖入口放在 Scene API 层（`setState` 的 options 参数）。
2. Scene 只存状态声明与元数据，不拥有元素实例。
3. 运行时补全过渡态，不回写历史场景。
4. 正放/倒放使用同一套规则，保证方向可逆。
5. 响应式规范下，方向偏移 `distance` 仅支持百分比字符串。

---

### 二、API 形态

#### entrance

```js
scene.setState(element, state, {
  entrance: true
})
```

```js
scene.setState(element, state, {
  entrance: {
    from: "bottom",
    distance: "5%"
  }
})
```

#### exit

```js
scene.setState(element, state, {
  exit: true
})
```

```js
scene.setState(element, state, {
  exit: {
    to: "top",
    distance: "4%"
  }
})
```

---

### 三、方向关键字

支持关键字：`"top" | "bottom" | "left" | "right"`

- `distance` 默认值为 `"4%"`
- `distance` 仅接受百分比字符串（如 `"4.5%"`）
- 不支持 px 与裸数字

---

### 四、语义规则

#### 1) 正放（forward）

- 元素首次出现：使用 `entrance.from`
- 元素离开目标场景：使用 `exit.to`

#### 2) 倒放（backward）

- 元素首次出现：倒放 `exit.to`
- 元素离开目标场景：倒放 `entrance.from`

#### 3) 默认可见性

当覆盖状态未显式声明 `opacity` 时，默认补 `opacity: 0`。

---

### 五、推荐写法

```js
scene2.setState(
  subtitle,
  {
    layout: { left: "50%", top: "56%", anchorX: 0.5, anchorY: 0.5 },
    opacity: 1,
  },
  {
    entrance: { from: "bottom", distance: "5%" },
    exit: { to: "bottom", distance: "5%" },
  }
)
```

---

### 六、与继承缓存的关系

- Scene 最终态由 resolved cache 计算。
- `entrance/exit` 在解析 `fromState/toState` 时基于最终态生效。
- `removeState` 会阻断继承，从而影响是否触发 entrance/exit 分支。

---

### 七、非目标范围（第一版）

1. 不做 timeline keyframe 系统。
2. 不做复杂缓动 DSL。
3. 不做编译期展开。
