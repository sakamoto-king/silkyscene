## 运行时对象与状态模型

SilkyScene 的核心是一套清晰的对象模型，其中 Element、Scene、Renderer 各司其职。理解这个模型对于正确使用 SilkyScene 至关重要。

### 生命周期与所有权

**Element** 的生命周期由 Presentation（演示文稿）管理。一个 Element 一旦被加入 Presentation，就会存在于整个演示的生命周期内。Element 可以在多个 Scene 之间使用，其状态在不同 Scene 中可以不同。

**Scene** 是状态的集合，不拥有 Element。每个 Scene 本质上是一张"元素状态快照"，记录某一时刻每个 Element 应该如何表现。Scene 与 Scene 之间可以共享同一批 Element，只要它们分别为这些 Element 定义不同的状态。

**Presentation** 是容器与编排器。它保持 Element 池、管理 Scene 列表、驱动场景切换与时间线更新。一个 Presentation 对应一个演示实例，可能展示在网页某个容器中。

### Element 的数据结构

Element 是纯数据对象，包含八层属性：

1. **基础信息**：id（唯一标识）、name（易读名称）、type（类型标签）、visible（可见性）、locked（锁定状态）
2. **层级结构**：parent（父元素引用）、children（子元素数组），支持嵌套和树形关系
3. **布局描述**：layout 对象，用相对坐标描述元素相对于父容器的位置与尺寸（支持 mode:"relative"|"absolute"|"flex"，left/top/width/height 可以是像素、百分比或 "auto"，anchorX/anchorY 定义旋转与缩放的锚点）
4. **变换数据**：transform 对象，用平台无关的格式描述缩放、旋转、平移（x、y、scaleX、scaleY、rotation），便于多平台渲染
5. **视觉属性**：opacity（透明度）、zIndex（堆叠顺序）、blendMode（混合模式）
6. **计算结果**：computed 对象，存放最终的绝对坐标与尺寸（x、y、width、height），由 Renderer 计算并赋值
7. **运行时状态**：dirty 标志（标记需要重新计算），mounted（已挂载标志），支持增量渲染优化
8. **渲染器引用**：Renderer 使用 WeakMap 等间接方式维护 Element 到 DOM 的映射，Element 本身不包含 DOM 字段

这样的分层设计使得 Element 可以独立定义，不依赖特定的渲染后端。DOM、Canvas、WebGL 都可以通过 Renderer 来消费这些属性。

### Scene 的状态管理

Scene 通过 states 字典维护每个 Element 的状态快照。当调用 `scene.setState(element, { layout: { left: "50%" } })` 时，只有这个具体属性被记录，而 Element 本体不受影响。

多个 Scene 中的同一个 Element 可以有不同的状态：Scene A 让它居中，Scene B 让它靠左。当演示在 Scene 之间切换时，Renderer 会根据新的 Scene 状态重新计算 Element 的 computed，从而驱动动画或跳转。

### 计算流程

当 Scene 发生变化时（通常是用户点击或自动播放切到新场景），运行时执行以下流程：

1. **状态解析**：Renderer 读取新 Scene 中该 Element 的状态
2. **布局计算**：Layout Engine 将相对布局 + Element 自身属性计算为绝对坐标，赋值到 computed
3. **渲染输出**：Renderer 根据 computed 与 transform，生成 CSS transform 字符串或 Canvas 指令，更新页面

这个流程是**同步的、按需的、确定性的**。不会在帧循环中反复执行，也不会因为媒体播放或其他异步事件隐式触发。

### 多个 Renderer 与多平台支持

虽然当前主要使用 DOM Renderer，但架构从一开始就为多平台准备。Canvas Renderer、WebGL Renderer 甚至自定义渲染后端都只需要：

1. 理解 Element 的 computed 与 transform 数据格式
2. 拦截 Scene 换场的触发点，重新计算布局
3. 在帧循环中按照现有 transform 进行插值与渲染

这样，同一批 Element 就可以在不同渲染后端间无缝切换。

### 与传统 MVC 的类比

如果把 SilkyScene 看作数据管理模式，可以类比为：

- **Model**：Element（存放数据）
- **View State**：Scene（存放作用于某个 View 的状态）
- **Controller**：Presentation（协调生命周期与场景切换）
- **View**：Renderer（根据 Model + View State 生成输出）

这种分离使得单个 Element 可以在多个 Scene（多个 View State）中使用，而无需复制对象。
