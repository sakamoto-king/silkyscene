## 两帧渲染机制

两帧渲染用于保证入场动画稳定触发，避免浏览器把起始样式与目标样式合并在同一帧优化掉。

### 触发条件

当元素在目标场景首次出现，且解析出了 `fromState` 时，会进入两帧渲染流程。

### Phase 1（同步）

1. 将节点过渡时长设为 `0ms`。
2. 立刻应用 `fromState`（通常包含 `opacity: 0` 与方向偏移）。
3. 该阶段不做动画，仅用于把起始样式写入渲染树。

### Reflow

读取 `content.offsetHeight` 强制 reflow，确保浏览器已提交 Phase 1 的样式。

### Phase 2（下一帧）

1. 在 `requestAnimationFrame` 中恢复正常 transition。
2. 应用 `toState`。
3. 浏览器检测到样式差异，触发 CSS transition。

### 为什么需要两帧

如果同一帧内连续写入起始态和目标态，浏览器可能直接合并变更，导致动画不触发或首帧闪烁。

### 与语法糖的关系

- `entrance/exit` 负责生成 `fromState/toState`。
- 两帧机制只负责“如何稳定渲染 from -> to”。
- 二者职责分离：语义解析在 Presentation，执行在 Renderer。
